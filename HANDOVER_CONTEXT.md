# 현재 작업 맥락 및 다음 할 일 (HANDOVER CONTEXT)

이 문서는 집 컴퓨터에서 진행하던 작업을 회사 컴퓨터에서 이어서 할 수 있도록 현재 상황과 다음 스텝을 정리한 파일입니다.

## 1. 현재 문제점
- 사용자가 `▶ 에이전트 실행.bat` 파일을 실행했을 때, 단순히 `claude` CLI만 켜지고 멈춰있는 상태입니다.
- 자동화를 위해서는 사용자가 **슬랙 메시지 전체를 한 번에 붙여넣고**, 에이전트가 이를 읽어 `CLAUDE.md`의 규칙에 따라 파싱 및 엑셀 출력을 자동으로 진행해야 합니다.

## 2. 해결 계획 (다음에 진행할 작업)
단순한 `.bat` 파일만으로는 여러 줄의 슬랙 메시지 입력을 매끄럽게 받기가 어려우므로, **파이썬 입력 어댑터 스크립트(`start.py`)** 를 추가하여 흐름을 개선할 계획이었습니다.

### [추가할 파일: start.py]
터미널에서 여러 줄의 텍스트(슬랙 메시지)를 깔끔하게 입력받는 역할을 합니다.
1. 사용자에게 슬랙 메시지를 붙여넣고 마지막에 `END`를 입력하라고 안내합니다.
2. 입력받은 텍스트를 `output/00_raw_message.txt` 파일로 자동 저장합니다.
3. 파이썬 내부에서 `subprocess`를 통해 `claude -p "output/00_raw_message.txt 파일을 읽고 CLAUDE.md에 따라 처리해"` 명령어를 백그라운드로 실행하여 에이전트 워크플로우를 트리거합니다.

### [수정할 파일: ▶ 에이전트 실행.bat]
`claude`를 직접 실행하는 대신 새로 만든 `start.py`를 실행하도록 변경해야 합니다.
```bat
@echo off
chcp 65001 >nul
cd /d "%~dp0"

set PYTHONUTF8=1
call .venv\Scripts\activate.bat
python start.py

pause
```

## 3. 회사 컴퓨터에서 이어서 할 일
1. 회사 컴퓨터에서 이 깃허브 저장소를 `git clone` (또는 `pull`) 받습니다.
2. `start.py` 파일을 위 맥락에 맞게 생성(또는 AI에게 다시 요청)합니다.
3. `▶ 에이전트 실행.bat` 파일을 위와 같이 수정합니다.
4. `.bat` 파일을 실행하여 슬랙 메시지 입력을 테스트하고, 전체 흐름(json 추출 -> xlsx 생성)이 완벽하게 돌아가는지 확인합니다.
