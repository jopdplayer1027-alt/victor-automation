# 슬랙 워크플로우 메시지 파싱 스펙

## 구분선 패턴
`ㅡ` 문자 10개 이상 → 정규화 후 `---`로 통일

## 필드 레이아웃

### 인라인 복수 필드 (같은 줄, `|` 구분)
```
거래처명: {값} | 행사명: {값}
근무 시작일: YYYY-MM-DD | 근무 종료일: YYYY-MM-DD
행사 담당자 이름: {값} | 행사 담당자 직급: {값}
행사 담당자 연락처: {값} | 행사 담당자 이메일: {값}
```

### 다음 줄에 값이 오는 필드
```
업무 내용:
{값 (여러 줄 가능)}
근무복장:
{값}
고객 요청사항:
{자유 텍스트}
상담 특이사항:
{자유 텍스트}
```

### 혼합 패턴 (같은 줄 또는 다음 줄 모두 가능)
```
근무복장: {값}      ← 같은 줄
근무복장:           ← 다음 줄
{값}
```

## 근무시간/인원/성별 블록 파싱

이 필드는 완전 자유 텍스트. 아래 패턴들이 혼재함:

**패턴 A: 번호 목록**
```
1. 행사 개요
- 일시: 5월 21일 - 31일
- 근무 시간: 12시간

2. 요청 인원
- 팀장 1인
- 영어스탭 8인
```
→ 고수준 정보만 있음. 날짜별 상세 없음 → schedule 빈 배열, 에스컬레이션 필요.

**패턴 B: 날짜별 상세**
```
20일 수요일
06:00 ~ 10:00 스탭 2
20:00 ~ 24:30 팀장 1 / 스탭 4

21일 목요일
팀장 1 09:00 ~ 23:00
영어스탭 7 09:00 ~ 22:30
```
→ 날짜별 schedule 배열로 추출 가능.

**패턴 C: 기간별 묶음**
```
5월 21일(목) ~ 5월 31일(일): 영어스탭 10, 중국어스탭 2
+5월 21일(목) ~ 5월 23일(토): 일반스탭 3 추가
```
→ 날짜 범위 풀어서 각 날짜별 schedule 항목 생성.

## 시간 표기 정규화

| 원문 | 정규화 |
|------|--------|
| `06:00 ~ 10:00` | start: "06:00", end: "10:00" |
| `22:00 - 익일 02:00` | start: "22:00", end: "익일 02:00" |
| `20:00 ~ 24:30` | start: "20:00", end: "익일 00:30" |
| `09:00-22:30` | start: "09:00", end: "22:30" |

## 수정 메시지 키워드
메시지 내 감지 시 `is_amendment: true`:
`수정사항`, `수정`, `변경`, `변동`, `변동사항`
