# 인원요청서 자동 생성 에이전트

## 역할
영업팀 슬랙 워크플로우 "상담하기" 메시지를 입력받아 표준 인원요청서 텍스트(txt)와 운영 시트(xlsx)를 자동 생성한다.
사용자: 섭외팀 직원 | 실행 환경: Claude Code CLI (Windows 11)

---

## 실행 전 확인

`output/` 디렉터리에 중간 산출물이 있으면 해당 단계를 건너뛰고 이어서 시작한다.

| 존재하는 파일 | 건너뛰는 단계 |
|---|---|
| `output/00_raw_message.txt` | [0] 입력 정규화 |
| `output/01_extracted.json` | [0], [1] 필드 추출 |
| `output/02_requirement.txt` | [0]~[3] 양식 생성 |

처음부터 다시 실행하려면: `output/` 내 파일 삭제 후 재실행.

---

## 워크플로우

### [0] 입력 어댑터

**입력 판별**:
- 슬랙 메시지 텍스트(붙여넣기) → 그대로 정규화
- 슬랙 링크(https://...) → v2 미지원. "텍스트를 직접 붙여넣어 주세요" 안내

**정규화 작업**:
1. 구분선(`ㅡ` 10개 이상) 표준화 → `---`
2. 앞뒤 공백·빈 줄 정리

**수정 케이스 감지**:
- 메시지 앞부분(첫 5줄) 또는 전체에서 `수정사항|수정|변경|변동|변동사항` 키워드 확인
- 감지되면: `output/` 내 `{거래처명}_{행사명}*.xlsx` 탐색
  - 파일 있음 → `.claude/skills/xlsx-writer/scripts/detect_amendment.py` 실행 → 기존 시트 탭명에 `_수정전` 추가
  - 파일 없음 → "기존 파일을 찾지 못했습니다. 신규로 처리합니다." 안내

**저장**: `output/00_raw_message.txt`

**실패 처리**: 텍스트에서 필드 레이블이 전혀 감지되지 않으면 사용자에게 형식 확인 요청.

---

### [1] 메시지 파싱 및 필드 추출

**처리 주체**: LLM(나)

**슬랙 메시지 포맷 규칙**:
```
거래처명: {값} | 행사명: {값}
근무 시작일: YYYY-MM-DD | 근무 종료일: YYYY-MM-DD
근무시간 / 인원 / 성별:
{자유 텍스트}
장소(실내/외): {값}
---
업무 내용: {값}
근무복장: {값}
복장 제공 여부: {값}
식사 제공 여부: {값}
---
고객 요청사항: {자유 텍스트}
상담 특이사항: {자유 텍스트}
---
문의경로: {값} | 유입경로: {값} | 링크 발송: {값}
---
행사 담당자 이름: {값} | 행사 담당자 직급: {값}
행사 담당자 연락처: {값} | 행사 담당자 이메일: {값}
```

**파싱 시 주의사항**:
- `근무시간/인원/성별` 블록은 자유 텍스트 → LLM이 날짜별·역할별 인원 튜플로 정규화
- 초기 메시지는 총괄 인원만 있을 수 있음 (날짜별 상세 없을 수 있음)
- 부가 일정(세팅/사전교육 등) 단서를 고객 요청사항·상담 특이사항에서 탐색

**출력 JSON 스키마** (`output/01_extracted.json`):
```json
{
  "client": "str",
  "event_name": "str",
  "start_date": "YYYY-MM-DD",
  "end_date": "YYYY-MM-DD",
  "venue": "str",
  "work_description": "str",
  "uniform": "str",
  "uniform_provided": "str",
  "meal_provided": "str",
  "client_requests": "str",
  "remarks": "str",
  "contact_channel": "str",
  "inflow_channel": "str",
  "event_manager": {
    "name": "str",
    "title": "str",
    "phone": "str",
    "email": "str"
  },
  "is_amendment": false,
  "schedule": [
    {
      "date": "YYYY-MM-DD",
      "label": "str (빈 문자열 또는 세팅/오픈/철수 등)",
      "staff": [
        {
          "role": "str (원문 그대로, 예: 팀장, 영어스탭, 중국어스탭, 일반스탭(재고관리))",
          "count": 1,
          "start": "HH:MM",
          "end": "HH:MM 또는 익일 HH:MM"
        }
      ]
    }
  ],
  "additional_schedules": [
    {
      "type": "세팅|사전교육|리허설|기타",
      "date": "YYYY-MM-DD 또는 미확정",
      "note": "str"
    }
  ]
}
```

**스키마 검증**: JSON 저장 후 반드시 실행:
```
python .claude/skills/memo-parser/scripts/validate_schema.py
```
실패하면 1회 재시도 후에도 실패 시 에스컬레이션.

**저장**: `output/01_extracted.json`

---

### [2] 필수 필드 검증 + 부가 일정 확인

**규칙 검증 (스크립트)**: validate_schema.py 결과 활용
- 날짜 형식 YYYY-MM-DD
- 시간 형식 HH:MM
- 인원수 > 0
- end_date >= start_date

**의미 검증 (LLM)**:
- schedule 항목이 1개 이상인지
- 날짜별 스케줄이 비어있고 메시지에도 상세 정보가 없으면 → 에스컬레이션

**부가 일정 판단**:
- additional_schedules 단서 있음 → 사용자에게 확인
- 단서 없음 → "세팅·사전교육 일정이 있나요?" 1회 확인

**에스컬레이션 규칙 (중요)**:
모든 확인 항목을 **한 번에** 번호 목록으로 제시. 절대 하나씩 묻지 말 것.

```
다음 항목들을 확인해 주세요:
1. [날짜별 스케줄 없음] 날짜별 상세 인원/시간 정보를 알려주세요.
2. [부가 일정] "전일 세팅" 언급이 있습니다. 세팅 날짜·시간을 알려주세요.
3. [인력구분] "VIP 안내" 처음 보는 표기입니다. 그대로 사용할까요?
```

### [3] 표준 양식 텍스트 생성

**처리 주체**: LLM + requirement-formatter 스킬

**스킬 참조**: `.claude/skills/requirement-formatter/references/output_format_template.md`

**출력 구조**:
- 상단: 기본정보 블록 (거래처, 행사명, 일정, 장소, 담당자 등)
- 하단: 날짜별 인력 스케줄 표 (인력구분별 행, 날짜별 열)

**저장**: `output/02_requirement.txt`

### [4] 양식-JSON 일치 검증

- JSON의 모든 핵심 값이 양식 텍스트에 반영되어 있는지 자기검증
- 임의로 추가된 정보 없는지 확인
- 실패 시 1회 재생성 후 에스컬레이션

### [5] 출력 채널 결정

**v1 (현재)**: 항상 xlsx 로컬 파일 생성.
사용자가 "구글 시트로" 요청해도 "v2에서 지원 예정"이라고 안내하고 xlsx 생성.

## 인력구분 처리 원칙

- 슬랙 메시지 원문 표기 그대로 보존 (임의 표준화 금지)
- 예: "영어스탭", "중국어 스탭", "일반스탭(재고관리)", "캐셔 서포트" 모두 그대로 사용
- 처음 보는 인력구분이 나와도 사용자 확인 없이 그대로 씀 (단, 에스컬레이션 목록에 표기)
